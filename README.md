# Leatherback Waypoint Navigation for Isaac Lab

**Developed by Nepher Team**

## Overview

This project implements a **waypoint navigation task** for the Leatherback wheeled robot in Isaac Lab. The task trains a policy to navigate the robot through a sequence of 3D waypoints using low-level throttle and steering control.

## Installation

1. Install Isaac Lab following the [installation guide](https://isaac-sim.github.io/IsaacLab/main/source/setup/installation/index.html).

2. Install the extension in editable mode:

    ```bash
    python -m pip install -e source/leatherbacknav
    ```

3. Verify installation:

    ```bash
    python scripts/list_envs.py
    ```

## Usage

### Training

```bash
python scripts/rsl_rl/train.py --task=Nepher-Leatherback-WaypointNav-v0
```

### Playing/Testing

```bash
python scripts/rsl_rl/play.py --task=Nepher-Leatherback-WaypointNav-Play-v0 --checkpoint=/path/to/checkpoint.pt
```

### Testing with Random Actions


## envs-nav Integration

This project includes integration with the [envs-nav](../envs-nav/) framework, which provides standardized navigation environments with predefined terrains, obstacles, and waypoint configurations.

### Overview

The envs-nav integration automatically loads:
- **Terrain**: Predefined terrain meshes from the waypoint-benchmark-v1 environment
- **Waypoints**: Waypoint sequences generated by the scene configuration
- **Obstacles**: Static obstacles placed according to the scene preset
- **Robot spawn positions**: Initial robot positions from the scene config
- **Episode length**: Maximum episode duration from the scene preset

### Environment Variants

Two environment variants are available with envs-nav integration:

1. **`Nepher-Leatherback-WaypointNav-Envs-v0`**: Training environment with envs-nav scenes
2. **`Nepher-Leatherback-WaypointNav-Envs-Play-v0`**: Play/evaluation environment with fewer environments and no observation noise

### Usage

#### Training with envs-nav

```bash
python scripts/rsl_rl/train.py --task=Nepher-Leatherback-WaypointNav-Envs-v0
```

#### Playing/Testing with envs-nav

```bash
python scripts/rsl_rl/play.py --task=Nepher-Leatherback-WaypointNav-Envs-Play-v0 --checkpoint=/path/to/checkpoint.pt
```

#### Customizing the Scene

You can customize the scene by passing a configuration object:

```python
import gymnasium as gym
from leatherbacknav.tasks.manager_based.waypoint_nav.waypoint_nav_env_cfg_envs_nav import WaypointNavEnvCfg_EnvsNav

# Use default scene (scene 0)
env = gym.make("Nepher-Leatherback-WaypointNav-Envs-v0")

# Use a different scene
cfg = WaypointNavEnvCfg_EnvsNav(nav_scene=1)
env = gym.make("Nepher-Leatherback-WaypointNav-Envs-v0", cfg=cfg)

# Use a different environment ID
cfg = WaypointNavEnvCfg_EnvsNav(nav_env_id="waypoint-benchmark-v1", nav_scene=2)
env = gym.make("Nepher-Leatherback-WaypointNav-Envs-v0", cfg=cfg)
```

### Configuration

The envs-nav integration uses the `WaypointNavEnvCfg_EnvsNav` configuration class, which:

- Automatically loads the scene configuration from envs-nav
- Configures terrain, obstacles, and lights from the preset
- Sets up waypoint generation using the scene's `gen_waypoints()` method
- Adjusts episode length based on the scene preset
- Sets environment spacing to match the scene requirements

Key configuration parameters:
- `nav_env_id`: Environment ID from envs-nav (default: `"waypoint-benchmark-v1"`)
- `nav_scene`: Scene/scenario index (default: `0`)
- `use_envs_nav_waypoints`: Automatically set to `True` when using envs-nav config

See `source/leatherbacknav/leatherbacknav/tasks/manager_based/waypoint_nav/waypoint_nav_env_cfg_envs_nav.py` for implementation details.

## Environment Details

The Leatherback is a 4-wheeled rover-style robot with:
- **Throttle control**: Velocity control for all 4 wheels
- **Steering control**: Position control for front wheel knuckles (Ackermann-style steering)

The action space is 2D: `[throttle, steering]`

See the configuration files in `source/leatherbacknav/leatherbacknav/tasks/manager_based/waypoint_nav/` for full details including observations, actions, rewards, and termination conditions.
